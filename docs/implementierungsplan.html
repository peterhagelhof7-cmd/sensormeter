HEUTE Ausgangspunkt

Prototyp v0.2.0

wt32-eth01-dht11 · platformio
  • Ethernet-Verbindung (LAN8720), ein DHT11
  • Einfacher Webserver (/, /json)
  • Syslog (RFC 5424), SNMPv2c + Zabbix-Template
STUFE 1 Grundgerüst & Zustandsmodell
P0

Projektstruktur & Boot-Zustandsautomat

Alle Module (Gerüst)
  • PlatformIO-Projekt gegliedert nach DataManager / ConfigManager / NetworkManager / TimeManager / StorageManager
  • Zustandsmodell: Boot → Init → Network Check → Run / Fallback
  • Korrigierte Pinbelegung v1.2 übernehmen (IO32/IO33 I²C, IO15, IO4 DHT11)
Voraussetzung: — Ersetzt Prototyp
STUFE 2 Parallel — Netzwerk/Zeit & Konfiguration
P1

Netzwerk & Zeit

NetworkManager · TimeManager
  • Ethernet DHCP/statisch, WLAN-Fallback „installer“ nach 5 Min
  • NTP-Sync (de.pool.ntp.org): 60s nach Boot, alle 5h, nach Link-Up
  • Fehlerkette: NTP-Fail → DHCP nach 5 Min → Restore nach 3 Min
Voraussetzung: P0 Neu
P2

Konfigurationspersistenz

ConfigManager · LittleFS
  • config.xml laden/speichern, Default-Fallback
  • XML-Import/-Export, Settings-Passwort (Default „installer“)
Voraussetzung: P0 Neu
STUFE 3 Sensorik
P3

Sensor-Datenerfassung

DataManager · SensorTask
  • DHT11 intern (60s-Takt), DHT22 extern optional (Sensor 2, PRO-Variante)
  • Plausibilitätsprüfung, Ringpuffer 168 Stundenwerte / 7 Tage
Voraussetzung: P0, Sensor-2-Flag aus P2 Baut auf Prototyp auf
STUFE 4 Parallel — Ausgabe- und Schnittstellen-Module
P4

Display

DisplayTask · SSD1306
  • I²C auf IO32/IO33, Seitenwechsel alle 10s
  • Boot-Countdown 100→0 bis Netzwerk bereit
Vor.: P1, P3 Neu
P5

Webserver

WebServerTask · StorageManager
  • Hauptseite: Status, Chart.js-Graph, Syslog-Tabelle, CSV-Download
  • Einstellungsseite (passwortgeschützt), REST-API
  • OTA-Versionscheck (GitHub Releases API), Anzeige „Update verfügbar"
  • Update-Button: direkt aus GitHub-Release oder lokaler .bin-Upload
Vor.: P1, P2, P3 Ersetzt Prototyp
P6

SNMP v1

SNMP-Agent (read-only)
  • Umbau von v2c auf v1
  • Spezifizierte OID-Struktur .1.3.6.1.4.1.99999.x
  • Zabbix-Template (optional, zusätzlich) an neue OIDs anpassen
Vor.: P1, P2, P3 Ersetzt Prototyp
P7

Syslog

SyslogTask · UDP 514
  • Format an Spezifikation anpassen (Name│IPs│RSSI│Werte│Zeit│Uptime)
  • Sofortversand bei Netzwerk-/NTP-/Sensorfehlern
Vor.: P1, P3 Baut auf Prototyp auf
